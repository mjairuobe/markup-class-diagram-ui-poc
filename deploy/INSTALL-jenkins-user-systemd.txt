Einmalige Einrichtung auf dem Jenkins-Agenten (User = jenkins o. ä.)

1) Als root (Headless, damit /run/user/<uid> dauerhaft existiert):
   loginctl enable-linger <jenkins-user>

2) Als jenkins, im ausgecheckten Repo:
   bash scripts/jenkins-vite-install-user-unit.sh

3) Pipeline wie gewohnt; bei fehlender Unit meldet jenkins-vite-pipeline-ctl.sh die obige Anleitung.

Steuerung / Beobachtung (als jenkins, mit gesetztem XDG_RUNTIME_DIR oder aus Login-Session):
  export XDG_RUNTIME_DIR=/run/user/$(id -u)
  systemctl --user status markup-vite-devserver.service
  systemctl --user restart markup-vite-devserver.service
  systemctl --user stop markup-vite-devserver.service
  busctl --user monitor org.markup.vite.DevServer
  journalctl --user -u markup-vite-devserver.service -f

Hinweis: JENKINS_HOME und HOME sollten auf dasselbe Verzeichnis zeigen (üblich: /var/lib/jenkins),
damit %h in der Unit mit ~/.markup-vite-devserver zusammenpasst.
